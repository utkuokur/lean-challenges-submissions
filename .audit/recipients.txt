# Recipients for the lean-challenges audit archive.
#
# Each non-comment line is an `age` recipient. age supports both native
# X25519 recipients (`age1...`) and SSH public keys (`ssh-rsa` >= 2048
# bits, `ssh-ed25519`). The submission workflow runs:
#
#   age --recipients-file .audit/recipients.txt --output X.age X
#
# on every submission, so any holder of a corresponding private key can
# decrypt all past and future archived submissions. Add or remove
# recipients via PR; the validate-recipients workflow lints each
# non-comment line.
#
# To rotate keys: add the new recipient on its own line, land the PR,
# and from then on new submissions are encrypted to the union of old and
# new. Old ciphertexts remain decryptable by their original recipients
# only; re-encrypting historical entries is a manual operation, not done
# automatically.
#
